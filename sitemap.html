Неделя 1 — Сбор материалов и концепции

П 1.1
Практика 1, часть 1

Сбор материалов для коммерческого лендинга

Выбираем нишу, собираем мудборд, контент и референсы — всё, что нужно для начала дизайна.
▼

Результат практики

Файл в Figma с мудбордом (2-3 штуки), контентом для лендинга и референсами по каждому блоку. Это основа для создания концепций.

Задание:
  • Выбрать тему для сайта через Рандомайзер (запускай 1-2 раза, не ищи специально «интересную» тему)
  • Собрать 2-3 мудборда проекта. Один мудборд = один будущий концепт
  • Найти сайт, откуда брать контент, или сгенерировать текст через ChatGPT по структуре продающего лендинга ?

    Структура продающего лендинга (используй как основу):

    1. Первый экран — заголовок-оффер, подзаголовок, кнопка CTA
    2. О компании / продукте — кратко: кто вы и чем занимаетесь
    3. Преимущества — 3-6 ключевых причин выбрать вас
    4. Как это работает — этапы работы / процесс (3-5 шагов)
    5. Услуги / Тарифы — что предлагаете и за сколько
    6. Примеры работ / Портфолио — если применимо
    7. Отзывы — 3-4 отзыва клиентов
    8. FAQ — 4-6 частых вопросов с ответами
    9. Контакты / Форма связи

    Промпт для ChatGPT:

    Сгенерируй текст для продающего лендинга на тему «[ТВОЯ НИША]». Нужен текст для блоков: первый экран (заголовок + подзаголовок), о компании, 4 преимущества, этапы работы (4 шага), 3 тарифа, 4 отзыва, 5 вопросов FAQ, контакты. Стиль: продающий, лаконичный, без воды.
  • Собрать по 5-10 референсов для каждого блока
⚠ Для поиска референсов НЕ рекомендую на первые разы использовать Dribbble и Behance — там много некачественных работ и вы будете копировать чужие ошибки

П 1.2
Практика 1, часть 2

Разработка концепций для лендинга

Создаём 2-3 концепции первого экрана — каждая в стиле своего мудборда.
▼

Результат практики

2-3 концепции первого экрана лендинга (шапка + главный блок с оффером), каждая в визуально отличающемся стиле. Лучшая концепция станет основой для полного дизайна на следующей неделе.

Задание:
  • Разработать по концепции в стиле каждого мудборда. 1 мудборд = 1 концепция

Концепции — это варианты дизайна, которые отличаются стилем / цветом / композицией. Важно, чтобы концепции не выглядели как варианты одного дизайна — это должны быть принципиально разные визуальные подходы.

⚠ Для новичков: используйте вайрфреймы и структуру из файла ниже для упрощения

Неделя 2 — Дизайн всего лендинга

П 2
Практика 2

Разработка дизайна всего лендинга (десктоп)

Концепции готовы, стиль определён — пора дизайнить остальные блоки сайта.
▼

Результат практики

Полный десктоп-макет коммерческого лендинга в Figma — все блоки от шапки до подвала, выстроенные по сетке. Применять AutoLayout и компоненты не обязательно — это дизайн для Тильды, и вы сами будете его верстать.

Задание:
  • Разработать дизайн всего лендинга, всех блоков

Как подойти к дизайну: выбери лучшую концепцию (по обратной связи от наставника), дизайни блоки сверху вниз — от первого экрана к подвалу. Используй сетку из урока 2.3, следи за визуальным ритмом, сверяйся с методичкой «Элементы сайта».

Неделя 3 — Адаптив

П 3
Практика 3

Разработка адаптива для лендинга

Создаём версии лендинга для планшета и мобильного — чтобы сайт хорошо выглядел на всех устройствах.
▼

Результат практики

Адаптивные макеты лендинга для планшета (768px) и мобильного (375px). Весь дизайн полностью готов к вёрстке на Тильде.

Задание:
  • Сделать адаптив своего лендинга для планшета и мобильного

Рекомендуемые разрешения: Десктоп 1440px (уже готов), Планшет 768px, Мобильный 375px.
Важно: не просто сжимай элементы — перестраивай композицию. Колонки (2-3 на десктопе) на мобильном становятся одной. Шрифты уменьшай на 20-30%. Кнопки — не меньше 44px.

Неделя 4 — Вёрстка на Тильде

П 4
Практика 4

Вёрстка лендинга на Тильде

Переносим готовый макет на Тильду, добавляем анимации, настраиваем и публикуем сайт.
▼

Результат практики

Опубликованный коммерческий лендинг на Тильде с адаптивом и анимациями — первый кейс для портфолио! Также подключена политика конфиденциальности и оферта.

Задание:
  • Перенести десктоп-макет и адаптив на Тильду
  • Добавить анимации (2-3 типа достаточно — не перебарщивай)
  • Настроить сайт перед публикацией: политика конфиденциальности и оферта

Советы: используй Zero Block для кастомных блоков. Начинай с десктопной версии, затем настрой адаптив в редакторе Тильды. Картинки экспортируй из Figma в PNG или SVG.
